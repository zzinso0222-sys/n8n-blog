### 서론  
AI 모델이 우리 일상에 깊이 스며들면서, 어떤 모델이 자유롭게 공개되어 있는지, 또 어떤 방식으로 사용해야 하는지 헷갈리는 경우가 많습니다. 오픈소스와 상용 라이선스, 그리고 각각의 이용 조건을 정확히 구분하면 개발 비용을 절감하고 법적 리스크를 피할 수 있습니다. 이 글에서는 AI 모델을 공개와 이용 방식별로 구분하는 핵심 포인트를 3가지 기준으로 정리합니다.

### 본론  

#### 1. 모델 소스와 라이선스 유형  
AI 모델은 크게 **오픈소스**와 **상용(클로즈드)** 으로 나뉩니다.  
- **오픈소스**는 GitHub, Hugging Face 등에서 공개되며, Apache 2.0, MIT, GPL 등 명시된 라이선스가 적용됩니다. 라이선스에 따라 재배포·수정·상업적 이용이 가능하지만, GPL 같은 강한 카피는 파생 코드에 동일 라이선스를 적용해야 합니다.  
- **상용 모델**은 OpenAI, Anthropic, Google 등 기업이 제공하는 API 형태로, 사용 약관(TOS)과 가격 정책에 따라 제한됩니다. 보통 API 호출당 비용, 사용량 상한, 데이터 소유권 등이 명시됩니다.  

#### 2. 공개 범위와 배포 방식  
- **전체 코드 공개**: 모델 아키텍처, 가중치, 학습 스크립트까지 모두 제공돼 로컬 환경에 직접 설치할 수 있습니다. 예시: LLaMA 2, Mistral 7B.  
- **가중치만 공개**: 모델 구조는 공개하지만 학습된 가중치는 API 혹은 제한된 다운로드 링크로만 제공됩니다. 예시: GPT‑4, Claude 3.  
- **API 기반 서비스**: 클라우드 엔드포인트에 모델을 호스팅하고, 개발자는 HTTP 요청만으로 모델을 호출합니다. 이 경우 실제 모델 파일은 접근할 수 없으며, 사용량·응답 지연·보안 정책을 고려해야 합니다.  

#### 3. 이용 조건 및 제한 사항  
- **상업적 이용 가능 여부**: Apache 2.0은 상업적 이용을 허용하지만, GPL은 제한적이거나 금합니다. 상용 API는 대부분 상업적 이용을 허용하되, 계약 조건에 따라 특정 산업(금융, 의료 등)에서는 제한될 수 있습니다.  
- **데이터 프라이버시**: 오픈소스 모델은 로컬에서 실행되므로 입력 데이터를 통제할 수 있지만, API를 사용할 경우 입력 데이터가 서비스 제공자에게 전송됩니다. GDPR·CCPA 등 규제 대상이라면 데이터 저장·전송 정책이 중요합니다.  
- **지원 및 유지보수**: 오픈소스는 커뮤니티와 문서만으로 지원되지만, 상용 API는 공식 지원·모니터링·보안 패치를 제공받을 수 있습니다. 운영 안정성을 우선시한다면 API 선택이 유리합니다.  

### 결론  
AI 모델을 선택할 때는 **소스 공개 여부**, **배포 방식**, **이용 조건**을 차례대로 검토하면 됩니다. 비용, 보안, 법적 요구사항을 모두 만족하는 모델을 찾는 것이 핵심이며, 오픈소스·상용 각각의 장단점을 명확히 이해하면 프로젝트 초기 단계에서 올바른 결정을 내릴 수 있습니다. 필요에 따라 두 방식을 조합(예: 오픈소스 모델을 API로 래핑)해 최적의 솔루션을 설계해 보세요.
